# Contributing to Chat Bot Platform (Megh EngageX)

Thank you for contributing! This guide will help you get started.

## Development Setup

### Prerequisites

- **Node.js** 20.x (use `nvm install 20` if you use nvm)
- **npm** >= 10
- **Docker** & **Docker Compose** (for containerized development)

### Quick Start

```bash
# Clone the repo
git clone https://github.com/rakzzm/Chat-Bot-Platform.git
cd Chat-Bot-Platform

# Run setup (creates .env, installs deps, generates secrets)
node setup.js

# Start with Docker (recommended)
docker compose up --build

# Or start manually
npm run dev:backend   # Terminal 1 — API on :4000
npm run dev:frontend  # Terminal 2 — UI on :8080
```

### Environment

Copy `.env.example` to `.env` and configure:

- `OPENROUTER_API_KEY` — Your OpenRouter API key for AI features
- `JWT_SECRET` — Auto-generated by setup.js (or set your own)
- `MONGO_URI` — MongoDB connection (default: local Docker instance)

## Architecture

### Backend (NestJS)

- **Modules**: Each domain (chat, user, nlp, cms, etc.) is a self-contained NestJS module
- **Database**: MongoDB via Mongoose with generic base repositories
- **Extensions**: Pluggable channels (web, console), helpers (LLM, NLU), and plugins
- **WebSocket**: Socket.IO with optional Redis adapter for horizontal scaling

### Frontend (Next.js 15)

- **Router**: Pages Router (not App Router)
- **UI**: Material UI 5 with React Flow for visual flow editor
- **State**: React Query for data fetching, Socket.io for real-time updates
- **i18n**: react-i18next with English and French locales

### Widget (Vite + React)

- Standalone embeddable chat widget
- Communicates with backend via Socket.IO
- Customizable themes and branding

## Development Workflow

1. **Create a branch** for your feature or fix
2. **Make changes** following existing code conventions
3. **Run checks** before committing:
   ```bash
   npm run lint        # Lint all packages
   npm run typecheck   # Type check all packages
   npm test            # Run tests (backend)
   ```
4. **Commit** with conventional commit messages:
   - `feat:` new features
   - `fix:` bug fixes
   - `perf:` performance improvements
   - `docs:` documentation
   - `chore:` tooling, config, etc.
5. **Open a PR** with a clear description of changes

## Testing

```bash
# Backend tests
cd packages/backend
npm test

# Specific test file
npm test -- --testPathPattern=block.service

# Coverage
npm run test:cov
```

## Adding Extensions

### New Channel

1. Create a directory under `packages/backend/src/extensions/channels/`
2. Implement the channel interface (see `web/` or `console/` as examples)
3. Export from `index.ts` with channel metadata

### New Helper (AI/NLP)

1. Create a directory under `packages/backend/src/extensions/helpers/`
2. Extend the base helper classes
3. Register in the extension system

### New Plugin

1. Create a directory under `packages/backend/src/extensions/plugins/`
2. Extend `BaseBlockPlugin` or `BaseEventPlugin`
3. Implement the required methods

## Docker

```bash
# Build and start all services
docker compose up --build

# Start in detached mode
docker compose up -d --build

# View logs
docker compose logs -f api
docker compose logs -f frontend

# Stop all services
docker compose down

# Reset data volumes
docker compose down -v
```

## Code Style

- **TypeScript** strict mode enabled
- **ESLint** + **Prettier** for formatting
- **No unused imports** or variables
- **Meaningful variable and function names**
